# Virust v0.5 Design - SSG, ISR, Caching & Performance

**Goal:** Add Static Site Generation (SSG), Incremental Static Regeneration (ISR), enhanced caching, and performance improvements to Virust

**Approach:** All-in-one release with core features first (SSG, ISR, caching), then streaming SSR as polish

**Status:** Design approved, ready for implementation planning

**Date:** March 6, 2026

---

## Overview

Virust v0.5 transforms from a server-side rendering framework into a full hybrid rendering platform. By adding static site generation, incremental regeneration, and intelligent caching, Virust will match and exceed Next.js's rendering capabilities while maintaining Rust's performance and type safety.

### What Makes v0.5 Special

- **True Hybrid Rendering**: SSR (runtime), SSG (build-time), and ISR (best of both)
- **Zero Configuration**: Code-based attributes, no config files needed
- **Performance First**: Parallel builds, intelligent caching, streaming output
- **Backward Compatible**: All v0.4 projects work without changes
- **Developer Experience**: Fast dev mode (SSR), powerful build mode (SSG)

---

## Architecture

### High-Level Request Flow

```
Development (virust dev):
  File Change → SSR (Bun) → Browser (instant feedback)

Production Build (virust build --ssg):
  1. Discover all routes with #[ssg] attribute
  2. For each route (in parallel):
     - Execute handler
     - Fetch data (async at build time)
     - Render component via Bun
     - Write HTML to dist/
  3. Generate ISR metadata (revalidation times)
  4. Output static files ready for deployment

Runtime (production server):
  1. Check if route has static HTML in dist/
  2. If yes and fresh → serve static file
  3. If expired or missing → SSR via Bun
  4. Apply cache headers from middleware
```

### Key Components

- **virust-build** (new crate) - SSG build orchestration with parallel rendering
- **CacheMiddleware** - HTTP caching layer with header support
- **SsgRegistry** - Tracks SSG routes and metadata
- **IsrManager** - Runtime static file serving and revalidation
- **#[ssg]** - Attribute macro for static generation
- **#[cache]** - Attribute macro for cache configuration

### Project Structure (New Output)

```
my-app/
├── dist/                      # NEW: SSG output
│   ├── index.html
│   ├── about/
│   │   └── index.html
│   ├── blog/
│   │   ├── first-post/
│   │   │   └── index.html
│   │   └── second-post/
│   │       └── index.html
│   └── .isr-metadata.json    # ISR metadata
├── api/                       # Existing routes

└── Cargo.toml
```

### 2. SSG Attribute Macro

```rust
// crates/virust-macros/src/ssg.rs
use proc_macro::{TokenStream, TokenTree};
use syn::{parse_macro_input, AttributeArgs, ItemFn};
use quote::quote;

#[proc_macro_attribute]
pub fn ssg(attrs: TokenStream, input: TokenStream) -> TokenStream {
    let args = parse_macro_input!(attrs as AttributeArgs);
    let input_fn = parse_macro_input!(input as ItemFn);

    // Parse arguments: #[ssg(revalidate = 60)]
    let revalidate = parse_revalidate_arg(&args);

    // Transform function to register as SSG route
    let fn_name = &input_fn.sig.ident;
    let fn_attrs = &input_fn.attrs;
    let fn_vis = &input_fn.vis;
    let fn_sig = &input_fn.sig;
    let fn_body = &input_fn.block;

    let expanded = quote! {
        #(#fn_attrs)*
        #fn_vis #fn_sig {
            #fn_body
        }

        // Register SSG route at compile time
        #[automatically_derived]
        impl virust_build::SsgRoute for #fn_name {
            const REVALIDATE: Option<u64> = #revalidate;
        }
    };

    TokenStream::from(expanded)
}

// Usage:
```

```rust
// crates/virust-runtime/src/cache.rs
use std::sync::Arc;
use std::time::SystemTime;
use tokio::sync::RwLock;
use axum::{extract::Request, middleware::Next, response::Response};
use http::{header, HeaderValue};

pub struct CacheMiddleware {
    store: Arc<RwLock<CacheStore>>,
}

pub struct CacheStore {
    entries: HashMap<String, CacheEntry>,
}

pub struct CacheEntry {
    pub body: Bytes,
    pub content_type: String,
    pub expires_at: Option<SystemTime>,
    pub etag: String,
    pub tags: Vec<String>,
}

#[derive(Clone, Default)]
pub struct CacheConfig {
    pub max_age: Option<u64>,           // seconds
    pub stale_while_revalidate: Option<u64>,
    pub stale_if_error: Option<u64>,
    pub tags: Vec<String>,
}

impl CacheMiddleware {
    pub fn new() -> Self {
        Self {
            store: Arc::new(RwLock::new(CacheStore::new())),
        }
    }

    /// Middleware that caches GET responses
    pub async fn cache_layer(
        req: Request,
        next: Next,
    ) -> Response {
        let method = req.method().clone();
        let uri = req.uri().clone();

        // Only cache GET requests
        if method != http::Method::GET {
            return next.run(req).await;
        }

        let cache_key = format!("{}:{}", method, uri);
        let store = self.store.read().await;

        // Check if we have a fresh cached response
        if let Some(entry) = store.get_fresh(&cache_key) {
            return build_cached_response(entry);
        }

        // No fresh cache, proceed to handler
        drop(store);
        let response = next.run(req).await;

        // Try to cache the response
        if let Some(cache_config) = extract_cache_config(&response) {
            let entry = CacheEntry::from_response(&response, cache_config);
            let mut store = self.store.write().await;
            let _ = store.insert(cache_key, entry);
        }

        response
    }
}

fn build_cached_response(entry: &CacheEntry) -> Response {
    Response::builder()
        .header(header::CONTENT_TYPE, &entry.content_type)
        .header(header::CACHE_CONTROL, format!("public, max-age={}", entry.max_age))
        .header(header::ETAG, &entry.etag)
        .header("X-Cache", "HIT")
        .body(entry.body.clone())
        .unwrap()
}

// Usage with attribute macro:

pub async fn get_todos() -> Json<Vec<Todo>> {
    // ...
}
```

```rust
// crates/virust-runtime/src/isr.rs
use std::collections::HashMap;
use std::path::PathBuf;
use std::time::SystemTime;

pub struct IsrManager {
    metadata: IsrMetadata,
    static_dir: PathBuf,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct IsrMetadata {
    pub routes: HashMap<String, RouteMeta>,
    pub generated_at: SystemTime,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct RouteMeta {
    pub path: String,
    pub file_path: String,
    pub generated_at: SystemTime,
    pub revalidate: Option<u64>, // seconds
    pub tags: Vec<String>,
}

impl IsrManager {
    pub fn new(static_dir: PathBuf) -> Result<Self> {
        // Load .isr-metadata.json
        let metadata_path = static_dir.join(".isr-metadata.json");
        let metadata: IsrMetadata = serde_json::from_str(
            &fs::read_to_string(metadata_path)?
        )?;

        Ok(Self {
            metadata,
            static_dir,
        })
    }

    /// Check if static file exists and is fresh
    pub async fn get_static_page(&self, path: &str) -> Option<Html<String>> {
        let route = self.metadata.routes.get(path)?;

        // Check freshness
        if let Some(revalidate) = route.revalidate {
            let age = SystemTime::now()
                .duration_since(route.generated_at)
                .ok()?;

            if age.as_secs() < revalidate {
                // Fresh - serve static file
                let html = fs::read_to_string(
                    self.static_dir.join(&route.file_path)
                ).ok()?;
                return Some(Html(html));
            } else {
                // Stale - trigger revalidation, serve stale in meantime
                let html = fs::read_to_string(
                    self.static_dir.join(&route.file_path)
                ).ok()?;
                self.trigger_revalidation(route);
                return Some(Html(html));
            }
        }

        // No revalidation - always serve static
        let html = fs::read_to_string(
            self.static_dir.join(&route.file_path)
        ).ok()?;
        Some(Html(html))
    }

    fn trigger_revalidation(&self, route: &RouteMeta) {
        // Spawn background task to regenerate
        tokio::spawn(async move {
            // Re-render page
            // Write to static dir
            // Update metadata
        });
    }

    /// Background task to regenerate stale pages
    pub async fn revalidation_task(&self) -> Result<()> {
        loop {
            tokio::time::sleep(tokio::time::Duration::from_secs(60)).await;

            for route in self.metadata.routes.values() {
                if let Some(revalidate) = route.revalidate {
                    let age = SystemTime::now()
                        .duration_since(route.generated_at)?;

                    if age.as_secs() >= revalidate {
                        self.trigger_revalidation(route);
                    }
                }
            }
        }
```

## Data Flow

### SSG Build Flow

```
1. User runs: virust build --ssg

2. SsgBuilder::discover_routes()
   - Scan api/ directory
   - Find functions with #[ssg] attribute
   - Extract: path, handler name, revalidate time
   - Build list: Vec<SsgRoute>

3. For each SsgRoute (parallel with tokio::spawn):

   a. Execute route handler
      - Call function: get_blog_post("first-post")
      - Handler runs async data fetching
      - Returns RenderedHtml with props

   b. Render component
      - Call Bun renderer with component name + props
      - Get HTML string output

   c. Write static file
      - Create dist/blog/first-post/index.html
      - Write HTML + hydration data

   d. Record metadata
      - Route path, generated timestamp
      - Revalidation interval
      - Tags for cache invalidation

4. Generate dist/.isr-metadata.json
   - All route metadata for runtime

5. Output build statistics
   - Pages built: 42
   - Routes with ISR: 15
   - Build time: 3.2s
```

### ISR Runtime Flow

```
1. Request: GET /blog/first-post

2. IsrManager checks:
   - Does dist/blog/first-post/index.html exist?
   - Is it fresh? (now - generated_at < revalidate)

3. Three outcomes:

   a. Fresh static file → return immediately
      - Add Cache-Control: public, max-age=60
      - Add ETag header
      - Return HTML

   b. Stale file → serve stale, regenerate in background
      - Return stale HTML immediately
      - Spawn background task to regenerate
      - Next visitor gets fresh version

   c. No file → SSR and cache
      - Render via Bun (SSR)
      - Save to dist/ for next time
      - Return HTML

4. Background revalidation task (every 60s):
   - Check all routes with revalidate times
   - Regenerate stale pages
   - Update .isr-metadata.json
```

### Cache Middleware Flow

```
Request → CacheMiddleware → Route Handler → CacheMiddleware → Response

1. Incoming request: GET /api/todos

2. Check cache:
   - Compute cache key from method + path
   - Lookup in memory store
   - Check freshness (max_age)

3. If cached and fresh:
   - Add Cache-Control header
   - Add X-Cache: HIT
   - Return cached response immediately

4. If not cached or stale:
   - Pass to route handler
   - Get response
   - Store in cache with expiry
   - Add X-Cache: MISS
   - Return response

5. Cache headers generated from #[cache] attribute:
   - max_age = 300 → Cache-Control: public, max-age=300
   - stale_while_revalidate = 60 → must-stale validation
```

### Streaming SSR Flow (Polish Phase)

```
1. Request: GET /blog (with streaming enabled)

2. Render component in chunks:
   - <html><head> → send immediately
   - </head><body> → send
   - <nav>...</nav> → send
   - Wait for async data →
   - <main>{posts}</main> → send
   - </body></html> → send

3. Browser renders progressively:
   - Shows layout immediately
   - Fills in content as it arrives
   - Better perceived performance

4. Implementation:
   - Use Transfer-Encoding: chunked
   - Bun renderer streams output
   - Axum body stream
```

## Error Handling

### SSG Build Errors

**Strategy:** Continue on error, log failures, partial success

```rust
pub async fn build(&self) -> Result<SsgStats> {
    let tasks: Vec<_> = self.routes.iter()
        .map(|route| self.build_route(route))
        .collect();

    let results = futures::future::join_all(tasks).await;

    let (successful, failed): (Vec<_>, Vec<_>) =
        results.into_iter().partition(Result::is_ok);

    // Log failures but don't fail entire build
    for error in &failed {
        eprintln!("❌ Failed to build {}: {}", error.route, error.msg);
    }

    if successful.is_empty() {
        anyhow::bail!("All pages failed to build");
    }

    Ok(SsgStats {
        pages_built: successful.len(),
        pages_failed: failed.len(),
        // ...
    })
}
```

### ISR Runtime Errors

**Strategy:** Graceful degradation to SSR

```rust
pub async fn get_static_page(&self, path: &str) -> Option<Html<String>> {
    match read_static_file(path) {
        Ok(html) => Some(html),
        Err(e) => {

        }

}
```

### Cache Errors

**Strategy:** Cache is optional, never break requests

```rust
pub async fn cache_layer(req: Request, next: Next) -> Response {
    // If cache store fails, just pass through
    let cached = match self.store.get(&key) {
        Ok(entry) => Some(entry),
        Err(e) => {
            eprintln!("Cache lookup failed: {}", e);
            None
        }
    };

    let response = next.run(req).await;

    // Try to cache, but ignore failures
    let _ = self.store.insert(key, response.clone());

    response
}
```

## Technical Decisions

### Why Code-Based Configuration?

**Decision:** Use attributes (`#[ssg]`, `#[cache]`) instead of config files

**Reasons:**
- Consistent with existing Virust patterns (`#[get]`, `#[post]`)
- Configuration lives with the code it affects
- No new file formats or parsing logic
- Easier to maintain as codebase grows
- Better IDE support (autocomplete, refactoring)

### Why Next.js-Style Output?

**Decision:** Use nested `index.html` files (e.g., `dist/blog/post/index.html`)

**Reasons:**
- Familiar to Next.js developers
- Server-ready (works with any web server)
- Clean URLs (no `.html` extensions)
- Supports nested routes naturally
- Easy deployment to Vercel, Netlify, etc.

### Why Time-Based ISR Only?

**Decision:** Support time-based revalidation, defer on-demand/tag-based

**Reasons:**
- Simpler implementation for v0.5
- Covers most use cases (blogs, docs, marketing pages)
- Can add on-demand later without breaking changes
- Reduces complexity and testing surface

### Why Dev Mode Skips SSG?

**Decision:** `virust dev` uses SSR only, `virust build --ssg` generates static

**Reasons:**
- Fast iteration in development
- Accurate production behavior in builds
- Clear separation of concerns
- No confusion about which mode is active

## Risks & Mitigations

### Risk: Bun Process Stability

**Concern:** SSG builds require stable Bun renderer for many pages

**Mitigation:**
- Restart Bun renderer every N pages
- Add health checks between builds
- Graceful error handling and retries
- Log Bun process health

### Risk: Build Time Complexity

**Concern:** Parallel builds may have race conditions or deadlocks

**Mitigation:**
- Use message passing, not shared state
- Limit parallelism with `--jobs` flag
- Comprehensive testing of concurrent builds
- Clear documentation of limitations

### Risk: Cache Stampede

**Concern:** Many requests for expired cache simultaneously

**Mitigation:**
- Serve stale while revalidating
- Single flight for cache refresh
- Configurable `stale_while_revalidate`
- Lock-free cache implementation

### Risk: Breaking Changes

**Concern:** New features might break existing v0.4 projects

**Mitigation:**
- All features are opt-in via attributes
- Comprehensive test suite with v0.4 templates
- Beta testing period
- Clear migration guide
