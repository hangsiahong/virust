# Virust v0.1 Design Document

**Date:** 2026-03-05
**Status:** Approved
**Version:** 0.1.0

---

## Overview

Virust is a Rust framework for **real-time, stateless backend services** optimized for AI-assisted development. v0.1 establishes the foundation with WebSocket-first communication, filesystem routing, macro-based RPC, and full TypeScript code generation.

**Key Goal:** Minimal boilerplate. Developers only write business logic.

---

## Architecture

### System Components

```
┌─────────────────────────────────────────────────────────┐
│                    virust-cli                           │
│  (init | dev | build) + Filesystem route discovery      │
└──────────────┬──────────────────────────────────────────┘
               │
┌──────────────▼──────────────────────────────────────────┐
│                 virust-runtime                          │
│  HTTP server │ WebSocket server │ Route dispatch        │
│  ──► Discovers api/**/*.rs files at startup             │
└──────────────┬──────────────────────────────────────────┘
               │
┌──────────────▼──────────────────────────────────────────┐
│              virust-protocol                            │
│  Message types │ HTTP types │ JSON-RPC │ Errors         │
└──────────────┬──────────────────────────────────────────┘
               │
┌──────────────▼──────────────────────────────────────────┐
│               virust-macros                             │
│  #[ws] #[get] #[post] #[put] #[delete]                  │
│  ──► Handler registry │ HTTP+WS │ TS generation         │
└─────────────────────────────────────────────────────────┘
```

**Data Flow:**
1. CLI spawns runtime
2. Runtime discovers and loads handlers (via macro registry)
3. WebSocket/HTTP connections accepted
4. JSON-RPC messages dispatched to handlers
5. Response sent back

**Build-Time Code Generation:**
- `#[ws]` macro expands to handler function + registers in global registry + writes TypeScript to `OUT_DIR`
- `build.rs` collects all TypeScript into `generated_types.ts`

## Components

### virust-protocol

**Purpose:** Define all shared types and message formats

**Core Types:**
```rust
pub struct RpcRequest {
    pub id: u64,
    pub method: String,
    pub params: Value,
}

pub struct RpcResponse {
    pub id: u64,
    pub result: Option<Value>,
    pub error: Option<RpcError>,
}

pub struct HttpRequest {
    pub method: HttpMethod,
    pub path: HashMap<String, String>,
    pub query: HashMap<String, String>,
    pub body: Option<Value>,
}

pub struct HttpResponse {
    pub status: u16,
    pub body: Option<Value>,
}

#[derive(Serialize)]
pub enum RpcError {
    NotFound(String),
    InvalidInput(String),
    Internal(String),
}
```

### virust-macros

**Purpose:** Generate handler code, maintain registry, output TypeScript

**Macro Expansion:**
```rust
// Input
#[ws]
async fn chat(msg: ChatMessage) -> ChatResponse { }

// Output (simplified)
pub fn chat_handler(req: RpcRequest) -> RpcResponse {
    let msg: ChatMessage = serde_json::from_value(req.params)?;
    let result = chat(msg).await?;
    Ok(to_response(result))
}

inventory::submit!(RouteEntry {
    path: "/chat",
    handler: chat_handler,
    route_type: RouteType::WebSocket,
})

// TypeScript output (to OUT_DIR/types.ts)
export function chat(msg: ChatMessage): Promise<ChatResponse>
```

**Provided Macros:**
- `#[ws]` - WebSocket RPC handler
- `#[get]`, `#[post]`, `#[put]`, `#[delete]` - HTTP method handlers
- `#[body]` - Extract JSON body
- `#[path("param")]` - Extract path parameter

## Filesystem Routing (Next.js Style)

```
myapp/
├ api/
│  ├ auth/
│  │  └ route.rs          → GET/POST/PUT/DELETE /api/auth
│  ├ chat/
│  │  └ route.rs          → GET/POST/PUT/DELETE /api/chat
│  └ users/
│     ├ [id]/
│     │  └ route.rs       → GET/POST/PUT/DELETE /api/users/:id
│     └ route.rs          → GET/POST/PUT/DELETE /api/users
```

**Route Discovery:**
- At startup (and on file changes in dev mode), runtime scans `api/` directory
- Finds all `route.rs` files
- Registers handlers based on directory structure and macro attributes

**Handler Example:**
```rust
// api/chat/route.rs

#[ws]
async fn chat(msg: ChatMessage) -> ChatResponse {
    ChatResponse { ok: true }
}

#[get]
async fn get_messages() -> Vec<Message> {
    vec![]
}

#[post]
async fn send_message(#[body] msg: SendMessage) -> Message {
    Message::new(msg.content)
}
```

## Data Flows

### HTTP Request Flow

```
Client: GET /api/users/123
    ↓
Router matches: /api/users/:id → api/users/[id]/route.rs → #[get]
    ↓
Extracts path params: { "id": "123" }
    ↓
Handler: get_user("123")
    ↓
Returns: User { id: 123, name: "Alice" }
    ↓
Serializes to JSON
    ↓
Response: 200 OK {"id":123,"name":"Alice"}
```

### WebSocket RPC Flow

```
Client: ws://localhost:3000
    ↓
WebSocket Upgraded
    ↓
Client sends: {"id":1,"method":"chat.send","params":{"message":"hello"}}
    ↓
Parses JSON-RPC, looks up "chat.send"
    ↓
Route Registry: "chat.send" → api/chat/route.rs::chat
    ↓
Deserializes params to ChatMessage
    ↓
Handler executes: chat(ChatMessage)
    ↓
Returns: ChatResponse { ok: true }
    ↓
Wraps in JSON-RPC response
    ↓
Server sends: {"id":1,"result":{"ok":true},"error":null}
```

### TypeScript Generation Flow (Build Time)

```
Developer: api/chat/route.rs with #[ws] handlers
    ↓ during cargo build
virust-macros #[ws] expansion:
  1. Generates handler wrapper code
  2. Registers in inventory::submit!()
  3. Writes to OUT_DIR/virust_types/chat.ts
    ↓
build.rs (virust-runtime):
  1. Collects all .ts files from OUT_DIR/virust_types/
  2. Concatenates to src/generated_types.ts
    ↓
Developer imports: import { chat } from './generated_types'
```

## Error Handling

### Structured Error System

```rust
#[derive(Debug, Serialize)]
pub enum ChatError {
    UserNotFound { user_id: String },
    MessageTooLong { length: usize, max: usize },
    InvalidChannel(String),
}

impl From<ChatError> for RpcError {
    fn from(err: ChatError) -> Self {
        match err {
            ChatError::UserNotFound { user_id } => RpcError {
                code: ERR_NOT_FOUND,
                message: format!("User not found: {}", user_id),
                details: Some(json!({ "user_id": user_id })),
            },
            // ... other variants
        }
    }
}
```

### Handler Error Propagation

```rust
#[ws]
async fn send_message(msg: SendMessage) -> Result<Message, ChatError> {
    if msg.content.len() > 1000 {
        return Err(ChatError::MessageTooLong {
            length: msg.content.len(),
            max: 1000,
        });
    }
    Ok(Message { ... })
}
```

### HTTP Status Code Mapping

```rust
ERR_NOT_FOUND     → 404
ERR_INVALID_PARAMS → 400
ERR_INTERNAL      → 500
```

### Development vs Production

**Dev mode:** Full stack traces, hints, error location
**Production:** Stripped details for security

## Implementation Approach

**Build-time code generation** with embedded templates:

1. TypeScript generated at compile time via `OUT_DIR`
2. Templates embedded in CLI binary with `include_str!`
3. File watching via `notify` crate with restart on change
4. Single binary distribution via `cargo install virust`

**Rationale:** Simplest implementation, fast compile times, easy debugging, well-established patterns.

---

## Success Criteria

v0.1 is successful when:

1. `virust init my-app --template chat` generates a working project
2. `virust dev` runs both frontend (Vite) and backend (Virust)
3. Editing `api/*.rs` files triggers auto-rebuild and restart
4. WebSocket RPC calls work with full type safety
5. TypeScript types stay in sync with Rust handlers
6. HTTP routes (GET/POST/PUT/DELETE) work correctly
7. Error responses are structured and helpful
8. All tests pass
