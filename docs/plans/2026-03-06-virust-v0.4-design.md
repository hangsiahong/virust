# Virust v0.4 Design - Hybrid Rendering with Bun

**Goal:** Add Next.js-style hybrid rendering (SSR, SSG, ISR, CSR) to Virust using Bun as the JavaScript runtime

**Approach:** Server-side rendering first (SSR), with foundation for SSG/ISR in future versions

**Status:** Design approved, ready for implementation planning

**Date:** 2026-03-06

---

## Overview

Virust v0.4 transforms from a JSON API backend into a full-stack framework with hybrid rendering capabilities. By integrating Bun as the JavaScript runtime, Virust will support:

1. **SSR (Server-Side Rendering)** - Render React/Vue components on the server
2. **React Server Components** - Async components with server-side data fetching
3. **Client Components** - Interactive components with `useState`, `useEffect`, etc.
4. **Type Safety** - TypeScript/TSX support out of the box

This is a foundational release - SSG and ISR will be added in v0.5+.

---

## Architecture Overview

### Request Flow (SSR)

```
1. Browser → http://localhost:3000/blog/post-1
2. Axum router matches route
3. Route handler returns RenderedHtml with component name + props
4. Virust calls Bun runtime:
   - Loads component from web/components/BlogPost.jsx
   - Executes async data fetching in component
   - Renders component to HTML string
5. Returns HTML + serialized props (for hydration)
6. Browser receives HTML, hydrates with React
```

### Key Components

- **virust-bun** (new crate) - Bun process management and IPC
- **virust-macros** - New `#[render_component]` attribute
- **Bun** - External JavaScript runtime for SSR
- **Component Registry** - Maps component names to file paths
- **Hydration System** - Embeds props in HTML for client-side hydration

### Project Structure

```
my-app/
├── src/
│   └── api/              # Rust API routes (existing)
├── web/
│   ├── components/       # React/Vue components (NEW)
│   │   ├── App.jsx       # Server component (async)
│   │   └── LikeButton.jsx  # Client component ('use client')
│   ├── pages/            # Page components (optional)
│   └── index.html        # Entry point
└── templates/            # Askama templates (optional)
```

## Data Flow & Request Lifecycle

### Complete SSR Request Flow

```
1. Browser Request
   ↓ GET /blog/post-123

2. Axum Router
   ↓ Matches #[get] #[render_component("BlogPost")]

3. Route Handler Execution
   ↓ async fn blog_post(#[path] id: String) -> RenderedHtml {
       RenderedHtml::with_props(json!({"id": id}))
     }

4. BunRenderer::render_component()
   ↓ a. Send IPC to Bun process:
      {"component": "BlogPost", "props": {"id": "post-123"}}

   ↓ b. Bun loads component:
      import('/app/web/components/BlogPost.jsx')

   ↓ c. Execute async component:
      const post = await fetch('/api/posts/post-123')...
      return <article>{post.title}</article>

   ↓ d. Render to HTML:
      renderToStaticMarkup(<BlogPost id="post-123" />)

   ↓ e. Return response:
      {"html": "<article>...</article>", "hydrationData": {...}}

5. Build HTTP Response
   ↓ <!DOCTYPE html>
      <html>
      <head><script src="/bun/client.js"></script></head>
      <body>
        <div id="root"><article>...</article></div>
        <script>__VIRUST_PROPS__ = {"id":"post-123",...}</script>
      </body>
      </html>

6. Browser Response
   ↓ - HTML displays immediately (SEO-friendly)
   ↓ - React hydrates #root
   ↓ - Page becomes interactive
```

### Client Component Detection

## Development Mode & Hot Reload

### Development Workflow

```bash
virust dev
```

**What Happens:**

1. **Rust Server** - Axum starts on :3000
2. **Bun Process** - Spawned for SSR
3. **File Watchers** - Monitor both Rust and component files
4. **HMR Server** - WebSocket for hot reloading

### File Watching

```
File Change Detection:
├── src/api/**/*.rs    → Restart Rust server (existing)
├── web/components/**  → Notify Bun, clear component cache
└── web/**             → Browser reload (existing HMR)
```

### Bun Component Cache

```rust
// BunRenderer maintains component cache
impl BunRenderer {
    pub async fn invalidate_cache(&self, component_name: &str) {
        self.send_to_bun(json!({
            "type": "invalidate",
            "component": component_name
        })).await;
    }
}
```

### HMR Flow

```javascript
// Client-side HMR
const ws = new WebSocket('ws://localhost:3000/__hmr');

ws.onmessage = (msg) => {
  const data = JSON.parse(msg.data);

  if (data.type === 'component-update') {
    // Component changed, hot reload
    import('/web/components/' + data.component + '.jsx')
      .then(() => console.log('Component reloaded'));
  }

  if (data.type === 'reload') {
    location.reload();  // Fallback
  }
};
```

## Error Handling & Edge Cases

### Component Loading Errors

```rust
impl BunRenderer {
    pub async fn render_component(&self, name: &str, props: Value) -> Result<RenderedOutput> {
        let response = self.send_to_bun(json!({
            "component": name,
            "props": props
        })).await?;

        if response["error"].is_string() {
            return Err(anyhow::anyhow!("Component error: {}", response["error"]));
        }

        Ok(RenderedOutput {
            html: response["html"].as_str().unwrap().to_string(),
            hydration_data: response["hydrationData"].as_str().unwrap().to_string(),
        })
    }
}
```

### Component Not Found

```javascript
// Bun renderer
try {
  const component = await import(componentPath);
} catch (error) {
  return { error: `Component not found: ${componentName}` };
}
```

### Async Component Errors

```jsx
// web/components/BlogPost.jsx
export default async function BlogPost({ id }) {
  try {
    const post = await fetch(`/api/posts/${id}`).then(r => r.json());
    return <article>{post.title}</article>;
  } catch (error) {
    return <Error message={error.message} />;
  }
}
```

### Fallback Strategies

1. **Component fails to load** → Show error boundary
2. **Data fetching fails** → Component handles internally
3. **Bun process crashes** → Auto-restart Bun process
4. **Timeout** → 5s limit, then return fallback HTML

### Bun Process Supervision

```rust
pub struct BunSupervisor {
    renderer: Option<BunRenderer>,
}

impl BunSupervisor {
    pub async fn ensure_running(&mut self) -> Result<&BunRenderer> {
        if self.renderer.is_none() || !self.renderer.as_ref().unwrap().is_alive() {
            self.renderer = Some(BunRenderer::new()?);
        }
        Ok(self.renderer.as_ref().unwrap())
    }
}
```

## API Surface & Developer Experience

### Route Handler API

```rust
// Simple SSR
#[get]
#[render_component("Home")]
async fn index() -> RenderedHtml {
    RenderedHtml::with_props(json!({}))
}

// With path parameters
#[get]
#[render_component("BlogPost")]
async fn blog_post(#[path] id: String) -> RenderedHtml {
    RenderedHtml::with_props(json!({ "id": id }))
}

// With query parameters
#[get]
#[render_component("SearchResults")]
async fn search(#[path] query: String) -> RenderedHtml {
    RenderedHtml::with_props(json!({ "query": query }))
}

// POST request (rare but possible)
#[post]
#[render_component("SuccessPage")]
async fn submit_form(#[body] form: FormData) -> RenderedHtml {
    RenderedHtml::with_props(json!({ "data": form }))
}
```
