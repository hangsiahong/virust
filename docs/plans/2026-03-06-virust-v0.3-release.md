# Virust v0.3 Release Notes

**Release Date:** 2026-03-06

## Overview

Virust v0.3 completes the filesystem-based routing system with full TypeScript support, automatic route discovery, and single-port development mode. This release transforms Virust from a proof-of-concept into a practical framework for building real-time backends.

## What's New

### 1. Parameter Extraction with `#[path]` and `#[body]`

Route handlers can now use declarative parameter attributes:

**Before (v0.2):**
```rust
#[post]
async fn create_todo(payload: String) -> String {
    let input: CreateTodoRequest = serde_json::from_str(&payload)?;
    // ...
}
```

**After (v0.3):**
```rust
#[post]
async fn create_todo(#[body] payload: CreateTodoRequest) -> TodoResponse {
    // payload is already deserialized
    // ...
}

#[get]
async fn get_todo(#[path] id: String) -> TodoResponse {
    // id is extracted from URL path
    // ...
}
```

**Benefits:**
- Less boilerplate code
- Compile-time type checking
- Better error messages (handled by Axum)
- Support for multiple parameters

### 2. Complete TypeScript Generation

TypeScript interfaces now include complete field definitions:

**Before (v0.2):**
```typescript
export interface CreateTodoRequest {
  // Field definitions will be added in future versions
}
```

**After (v0.3):**
```typescript
export interface CreateTodoRequest {
  title: string;
  description?: string | null;
}

export interface TodoResponse {
  id: string;
  title: string;
  description?: string | null;
  completed: boolean;
  created_at: number;
}
```

**Type Mapping:**
- `String` → `string`
- `i32/i64` → `number`
- `bool` → `boolean`
- `Option<T>` → `T | null`
- `Vec<T>` → `T[]`

Access generated types at: `http://localhost:3000/api/__types`

### 3. Automatic Route Discovery

Routes are automatically discovered from the `api/` directory at startup:

```
api/
├ chat/
│ └ route.rs          → Automatically registered as /api/chat
├ todos/
│ ├ route.rs          → GET /api/todos, POST /api/todos
│ └ [id]/
│   └ route.rs        → GET /api/todos/:id, PUT /api/todos/:id, DELETE /api/todos/:id
```

**Dynamic Routes:**
- Use `[param]` syntax for dynamic segments
- `api/users/[id]/route.rs` → `/api/users/:id`
- Multiple dynamic segments supported: `api/posts/[slug]/comments/[id]/route.rs`

**Benefits:**
- No manual route registration
- Filesystem maps 1:1 to URL structure
- Adding routes is just creating files

### 4. Single-Port Development Mode

Development is now simplified to a single command:

```bash
virust dev
```

This starts a server on port 3000 that serves:
- **Static files** from `web/` at `/`
- **API routes** from `api/` at `/api/*`
- **WebSocket** at `/ws`
- **TypeScript types** at `/api/__types`

### 5. Hot Module Replacement (HMR)

Frontend changes now reload automatically:

- Edit any file in `web/`
- WebSocket sends reload signal to browser
- Page refreshes automatically
- No manual refresh needed

Backend changes trigger:
- Automatic recompilation
- Server restart
- Ready to test immediately

## Breaking Changes from v0.2

### Route Handler Signature

Route handlers now use parameter extraction attributes:

**Old Syntax (deprecated but still works):**
```rust
#[post]
async fn create_todo(payload: String) -> String {
    // Manual JSON parsing required
}
```

**New Syntax (recommended):**
```rust
#[post]
async fn create_todo(#[body] payload: CreateTodoRequest) -> TodoResponse {
    // Automatic JSON parsing
}
```

### Development Server

**Old:**
```bash
# Terminal 1: Backend
cargo run

# Terminal 2: Frontend (with Vite)
cd web && npm run dev
```

**New:**
```bash
# Single command
virust dev
```

## Migration Guide

### Update Route Handlers

1. Add `#[body]` attribute to JSON body parameters
2. Add `#[path]` attribute to URL parameters
3. Change parameter types from `String` to actual struct types
4. Update return types to use structs instead of `String`

**Example:**
```rust
// Before
#[post]
async fn create_todo(payload: String) -> String {
    let req: CreateTodoRequest = serde_json::from_str(&payload)?;
    let response = TodoResponse { /* ... */ };
    serde_json::to_string(&response)?
}

// After
#[post]
async fn create_todo(#[body] payload: CreateTodoRequest) -> TodoResponse {
    TodoResponse { /* ... */ }
}
```

### Update Templates

If using v0.2 templates:

1. Update route handlers to use `#[path]` and `#[body]`
2. Remove Vite configuration if present
3. Update development workflow to use single `virust dev` command
4. Access TypeScript types at `/api/__types` instead of separate endpoint

## Known Limitations

### Multiple Path Parameters

Multiple path parameters in a single route are not yet supported:

**Not Yet Working:**
```rust
// api/posts/[post_id]/comments/[comment_id]/route.rs
#[get]
async fn get_comment(#[path] post_id: String, #[path] comment_id: String) -> Comment {
    // This will fail - use tuple syntax instead
}
```

**Workaround:**
```rust
#[get]
async fn get_comment(params: Path<(String, String)>) -> Comment {
    let (post_id, comment_id) = params.0;
    // ...
}
```

This will be improved in a future release to support:
```rust
#[get]
async fn get_comment(#[path] post_id: String, #[path] comment_id: String) -> Comment {
    // Coming soon
}
```

### Complex Generic Types

Very complex generic types may fall back to `any` in TypeScript:
```rust
pub struct ComplexType {
    pub data: HashMap<String, Vec<Option<Result<Item, Error>>>>,
}
// Generates: data: any;
```

### Recursive Types

Recursive type definitions generate `any` for recursive fields to prevent infinite loops.

5. **Filesystem Routing**: Intuitive structure matching URLs
